# 要件定義書 (Requirements.md) - ドラフト

## 1. プロジェクト概要
就労継続支援A型事業所の現場スタッフが、スマートフォンからの音声入力を起点として、ケース記録や日報をAIで自動生成・ペーパーレス化するためのプロトタイプツール。

## 2. ターゲットユーザー
- 就労継続支援A型事業所の現場スタッフ（支援員）
- 年齢層が高く、ITツールに不慣れな層（IT弱者）を含む

## 3. 鉄の掟（UI/UX・開発基準）
- **初期費用・維持費の最小化**: 無料枠や格安APIを徹底活用。
- **高齢者・IT弱者配慮**:
  - フォントサイズ: 18px以上（可読性重視）
  - タップ領域: 44px以上のボタン（押し間違い防止）
  - コントラスト比: 4.5:1以上（視認性確保）
  - 1画面1アクション: 迷わせないシンプルな導線
- **完全スマホファースト**: 片手で操作できるインターフェース。

## 4. 主要機能（MVPスコープ）
1. **音声入力・文字起こし**:
   - ボタン一つで録音開始。
   - 現場の話し言葉をリアルタイムまたはバッチでテキスト化。
2. **AI記録生成**:
   - 文字起こしされた断片的な情報を、適切な「ケース記録」や「日報」の形式に整形。
   - A型事業所特有の用語や文脈を考慮。
3. **内容確認・修正**:
   - 生成されたテキストを確認し、必要に応じて簡易修正。
4. **保存・出力**:
   - データベースへの保存。
   - 既存の紙運用に合わせた表示/印刷用プレビュー。

## 5. 非機能要件
- **セキュリティ**: 個人情報（利用者名等）の取り扱いに配慮（伏せ字化などの検討）。
- **保守性**: 社長1人でメンテナンス可能なシンプルなコードベース。
