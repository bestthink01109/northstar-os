# BtoB向け AIケアプラン作成支援ツール 要件定義書 (No-Code MVP版)

## 1. プロジェクト概要
新コンセプトである「初期費用ゼロ・少額」「社長1人での運用」を前提とし、外注費をかけずにノーコードツールとAI APIを組み合わせて、ケアマネジャー向け「音声入力→ケアプラン（原案）自動生成アプリ」のMVP（最低限のプロトタイプ）を構築する。

## 2. システム構成（ノーコード・ローコード戦略）
高額なシステム開発を避け、以下のSaaS・APIを連携させて構築する。

- **フロントエンド (UI/画面)**: `Glide` または `Bubble` または `FlutterFlow`。
  - スマートフォンのブラウザからアクセスし、マイクボタンで録音できるシンプルなUI。構築はAIサポートを受けながら1人で実施可能。
- **バックエンド・ロジック**: `Make (Integromat)` または `Zapier`。
  - 音声データの受け渡しと、生成されたテキストのデータベース保存を自動化。
- **AI・音声処理**: `OpenAI API (Whisper + GPT-4o)` または `Dify`。
  - Whisperで音声を高精度テキスト化し、専用にチューニングしたシステムプロンプト（RAG）を挟んでケアプランの指定フォーマット（第1表・第2表など）の形に出力させる。

## 3. コア機能要件
1. **音声録音＆送信**: スマホからヒアリング内容を録音。
2. **文字起こし＆定型フォーマット化**: AIが不要な会話を削り、ケアプランの項目に必要な要素を自動抽出。
3. **コピー＆ペースト用出力**: 既存の介護ソフト（ワイズマン、カイポケ等）にコピペしやすいテキスト形式で結果を画面表示。※最初はAPI連携など複雑なことはせず、「コピペツール」として割り切る（開発コストゼロ化のため）。

## 4. 開発フェーズ (1人で完結)
- **Phase 1 (プロンプト開発)**: まずは画面を作らず、ChatGPT（ブラウザ版）のカスタムGPTsとして「ケアプラン生成GPT」を作り、精度を検証する（**費用：0円**）。
- **Phase 2 (アプリ化)**: Glide等のノーコードツールで「ボタン1つの音声入力アプリ」として組み上げる。
- **Phase 3 (顧客テスト)**: 既存のコンサル・代行先の少数の顧客に無料または低額で使ってもらい、フィードバックを得る。
