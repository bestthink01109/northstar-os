# プロジェクト引継ぎコンテキスト：給与データ自動化エンジン（プロジェクトA）

最終更新：2026年4月15日
作成者：BUN_CEO / 開発責任者AI

このドキュメントは、現在本番運用可能となっている主要プロジェクト「給与データ自動化（給与らくだインポート連携）システム」の全体像、アプリ構成、および特殊なビジネスロジックについて、他のAIまたは開発者がスムーズに全容を把握できるようまとめた唯一の正式引継ぎ資料です。

---

## 1. システム全体図・概要（Overview）

現場の「Excel出勤簿」を読み込み、BUN_CEO指定の完全独自の労働規則（3層残業ロジック）を通して自動計算を行い、最終的に給与計算ソフト「給与らくだ」のインポート仕様に完全準拠したCSVデータを出力するシステムです。

### データフロー

```
[Excel出勤簿] → [ExcelParser] → [OvertimeCalculator] → [WeeklyAllocator] → [MonthlyAggregator] → [CSV出力]
  (入力)        (データ抽出)      (日次計算)            (週次振分)          (月次集計)            (出力)
```

- インプット：
  - 対象月の現場出勤簿ファイル（例：`出勤簿_202603.xlsx`）
  - Google Driveの指定フォルダから直接取得し処理を実行します。

- アウトプット：
  - `kyuyo_rakuda_202603.csv` 等のCSVフォーマット（21列、Shift-JIS/cp932エンコード）。
  - 出力先は `Google Drive/マイドライブ/Development/平野工業/` 配下に固定されています。

## 2. アプリケーション構成（Modules）

このアプリは1つの巨大なスクリプトではなく、役割ごとに分類された以下のPythonスクリプトファイル群で構成されています。

### 実行スクリプト（エントリーポイント）

- `export_rakuda_csv.py`
  - 役割：システムのメイン実行ファイル。インプットファイルの探索、コアエンジンの呼び出し、最終的なCSV出力を行います。
  - 特徴：「給与らくだ」が要求するフォーマット（社員氏名、タイムカード年月、休日出勤時間などを指定の順番で配置した21列のデータ）へ厳密にマッピングする処理を担っています。出力時の文字化け（機種依存文字の「﨑」など）を防ぐため、エンコードには `cp932` が採用されています。

### コアエンジン（ロジック処理の中心）

- `payroll_engine.py`
  - アプリの心臓部であり、以下の4つの主要クラスで構成される高度な算出エンジンです。
  1. `ExcelParser`：`openpyxl` を駆使し、複雑な出勤簿シートから必要な時刻や有給ステータス（全休、半休など）を正確に抽出する。
  2. `OvertimeCalculator`：各日の入退室基準で「日次の実働時間」や細かな休憩時間（10:00〜10:30, 12:00〜13:00, 14:00〜14:30）を差し引き「欠勤時間」を自動で算出する。
  3. `WeeklyAllocator`：後述する「BUN_CEO式 残業振り分けロジック」を適用し、法定内残業・法定外残業を数珠つなぎに相殺・算出する週次ベースの振り分け器。
  4. `MonthlyAggregator`：解析された全日次・週次データを月次として集計する。休日出勤時間や遅刻早退などの合算もここで行う。

### 補助スクリプト群

- `employee_master.py`
  - 全スタッフの「社員コード」および「氏名情報」を管理する辞書（マスター）機能。
  - 「平野珠美」氏などの特例労働条件者をプログラムが識別するための関数もここに集中しています。

- `test_engine.py` / `test_engine_mar.py`
  - 出勤簿の「手計算されたExcelの合計値」と「AIエンジンの計算値」を突き合わせ、100%一致するか検証するためのテストモジュール。
  - 重要：AIエンジンは時に「Excel側の人為的な記入漏れ（管理者のミス）」すらも自動検知し、指摘機能として報告できる精度を備えています。

## 3. 組み込み済みの BUN_CEO 特例ロジック

本システムには、一切妥協のない詳細な現場ルール（Business Logic）が実装されています。運用上絶対に改変してはならないコア技術です。

### 3-1. 勤務体系の前提条件（全ロジックの根幹）

本システムの全計算は、以下の勤務体系を前提に構築されています。

- 一般社員（平野珠美氏以外の全員）
  - 月曜〜金曜：所定労働時間 7時間（8:00〜17:00、休憩2時間）
  - 土曜日：通常出勤扱い 5時間（8:00〜15:00、休憩2時間）
  - 週所定労働時間：7時間 × 5日 ＋ 5時間 × 1日 ＝ 40時間（ピッタリ）
  - 日の法定労働時間上限：8時間

- 平野珠美氏（特例）
  - 月曜〜金曜：所定労働時間 8時間
  - 土曜日：通常出勤扱い（他社員と同様）
  - 週所定労働時間：8時間 × 5日 ＝ 40時間

### 3-2. 厳密な休憩時間の時間帯控除

ただ合計時間を引くのではなく、以下の3つの時間帯に社員が実際に就業していたかどうかで動的に休憩時間を差し引きます。

| 休憩区間 | 時間 | 控除時間 |
|---|---|---|
| 午前休憩 | 10:00〜10:30 | 0.5時間 |
| 昼休憩 | 12:00〜13:00 | 1.0時間 |
| 午後休憩 | 14:00〜14:30 | 0.5時間 |
| 合計 | | 2.0時間 |

例：8:00出勤〜17:00退勤の場合、拘束9時間 − 休憩2時間 ＝ 実働7時間

### 3-3. 自動欠勤の抽出

所定労働時間に対して、実際の労働時間と時間休の合計を比較し、少しでもショート（不足）している場合は、すべて自動的に「欠勤時間」として算出され、CSVの「遅刻早退時間」枠へ組み込まれます。

### 3-4. 時間有給（半日休）の動的切り替え

- 平野珠美氏（所定8時間）の場合：半日休 ＝ 4.0時間
-　1時間単位の有給はエクセル出勤簿の時間休（W列）に明記
- その他の社員（所定7時間）の場合：半日休 ＝ 3.5時間
- 1時間単位の有給はエクセル出勤簿の時間休（W列）に明記

### 3-5. 法定内残業（延長）と法定外残業、および週40時間の特例ロジック（最重要アルゴリズム）

本システムの核心であり、給与計算において最も複雑な「残業の2階層分類」と「週40時間上限」を処理する特例ルールです。以下の厳格な基準で判定されます。

#### 3-5-1. 原則となる1日の考え方

- 法定内残業（延長）：1日の所定労働時間を超え、法定労働時間（8時間）に達するまでの残業。
  - 平日（所定7時間）の場合：最大1時間が法定内残業の枠。
  - 土曜日（所定5時間）の場合：最大3時間が法定内残業の枠（5時間＋3時間＝8時間まで）。
  - CSVでは「延長時間」として出力されます。
- 法定外残業：1日8時間を超える分の残業。発生した日において即座に「法定外残業」として確定します。他の日の欠勤等との相殺は一切行いません。

#### 3-5-2. 週40時間の考え方（WeeklyAllocator の核心ロジック）

本システムにおける「週40時間」は、月曜〜土曜の6日間の所定労働時間の合計で判定します。日曜日は完全な休日であり、計算対象外です。

##### ケースA：基本の週（欠勤・有給・遅刻早退が一切ない場合）

```
月 7h ＋ 火 7h ＋ 水 7h ＋ 木 7h ＋ 金 7h ＋ 土 5h ＝ 週40時間（ピッタリ）
```

月〜土で所定労働時間が既に40時間に到達しているため、バッファ枠はゼロです。
したがって、この週において発生するすべての残業（平日の1時間以内であっても、土曜日の3時間以内であっても）は、日次では法定内（8時間以内）であったとしても、週40時間を超過するため、すべて法定外残業として判定されます。

結論：欠勤なしの通常週では、法定内残業（延長）は一切発生せず、残業はすべて法定外残業となる。

##### ケースB：欠勤・有給・遅刻早退がある週

月〜土の間に欠勤や有給（全休・半休）、遅刻早退が存在した場合、その不足分だけ週40時間に対するバッファ枠が動的に発生します。

例：水曜日に全休（所定7時間分の不足）が発生した場合

```
月 7h ＋ 火 7h ＋ 水 0h ＋ 木 7h ＋ 金 7h ＋ 土 5h ＝ 週33時間
→ 週40時間までのバッファ枠 ＝ 7時間
```

このバッファ枠7時間の範囲内で、他の出勤日に発生した残業は法定内残業（延長）として処理されます。ただし、以下の厳格な制約が適用されます。

- 各日において8時間を超えた分：即座に法定外残業で確定（欠勤との相殺は一切しない）。
- 各日において所定時間超〜8時間以内の分（平日なら1時間以内、土曜なら3時間以内）：週トータルのバッファ枠に収まる範囲で法定内残業（延長）として計上される。バッファ枠を超えた分は法定外残業へ昇格される。

例の続き（水曜全休、他の日に残業あり）：

```
月：8.5h勤務 → 所定超1h（延長候補）＋ 法定超0.5h（法定外確定）
火：8.0h勤務 → 所定超1h（延長候補）
木：7.5h勤務 → 所定超0.5h（延長候補）
金：8.0h勤務 → 所定超1h（延長候補）
土：7.0h勤務 → 所定超2h（延長候補）

延長候補の合計 ＝ 1 + 1 + 0.5 + 1 + 2 ＝ 5.5時間
バッファ枠 ＝ 7時間
5.5時間 ≦ 7時間 → 全額が法定内残業（延長）で確定
法定外残業 ＝ 0.5時間（月曜の8時間超過分のみ）
```

##### ケースC：例外パターン（平野珠美氏の場合）

所定労働時間が「1日8時間」のため、日次レベルで法定内残業の枠が存在しません（所定＝法定が同値）。1日8時間を超える労働はすべて即座に法定外残業判定となります。

#### 3-5-3. バグを防止する「日次ショート分の積和計算」

全体の週労働時間から単純に引き算するだけの処理にすると、ある日の欠勤（−7時間ショート）が、別の日の法定外残業（＋3時間など）と誤って打ち消し合ってしまい、本来支払うべき法定外残業代が減額されるという致命的なバグが起こります。

これを防ぐため、AIエンジンは以下の設計原則を厳守しています。

- 「日別の不就労時間（Unlabor）」と「法定外残業」を完全に分離して集計する。
- 不就労分（その日の所定時間に対する不足時間）だけを週40時間のバッファとして正確に足し合わせる。
- 法定外残業（8時間超過分）は日次で確定させ、週次の相殺計算には一切持ち込まない。

この「日次ショート分だけを積み上げてバッファとする」ロジックが `WeeklyAllocator` の根幹です。

## 4. CSV出力仕様（給与らくだインポートフォーマット）

21列構成、ヘッダーなし、cp932エンコーディング。列順は以下の通りです。

| 列番号 | 内容 | 備考 |
|---|---|---|
| A列 | 社員コード | employee_master.pyから取得 |
| B列 | 社員氏名 | employee_master.pyから取得 |
| C列 | タイムカード年 | 出勤簿の対象年 |
| D列 | タイムカード月 | 出勤簿の対象月 |
| E列 | 出勤日数 | 実出勤日数（有給休暇日を除く） |
| F列 | 有給日数 | 全休の日数 |
| G列 | 有給時間 | 半休の合計時間 |
| H列 | 特別休暇日数 | 通常0 |
| I列 | 欠勤日数 | 未出勤かつ有給でない日数 |
| J列 | 遅刻回数 | 通常0 |
| K列 | 早退回数 | 通常0 |
| L列 | 遅刻早退時間 | 自動欠勤抽出による不足時間合計 |
| M列 | 所定時間 | 月間の所定労働時間合計 |
| N列 | 延長時間 | 法定内残業の月間合計 |
| O列 | 法定外残業時間 | 法定外残業の月間合計 |
| P列 | 深夜残業時間 | 通常0 |
| Q列 | 深夜時間 | 通常0 |
| R列 | 休日出勤時間 | 日曜・祝日の出勤時間合計 |
| S列 | 休日深夜時間 | 通常0 |
| T列 | 代休日数 | 通常0 |
| U列 | 備考 | 空欄 |

## 5. 運用手順

1. 毎月の出勤簿ファイル（`出勤簿_YYYYMM.xlsx`）をGoogle Driveの `Development/平野工業/` フォルダに配置する。
2. `export_rakuda_csv.py` を実行する。
3. 同フォルダに `kyuyo_rakuda_YYYYMM.csv` が自動生成される。
4. このCSVファイルを「給与らくだ」ソフトにインポートする。

## 6. 技術的注意事項

- 文字コード：機種依存文字（﨑など）に対応するため、CSV書き出しは `cp932` を使用。
- 依存ライブラリ：`openpyxl`（Excel読み込み用）
- ロジック改変時の最重要確認ポイント：
  - `payroll_engine.py` の `WeeklyAllocator` における法定内/法定外の振り分けロジック
  - `OvertimeCalculator` の休憩時間控除ロジック（3区間の時間帯判定）
  - 平野珠美氏の特例判定（所定8時間、半休4.0時間）
