## 3. [OPS] 代行業務自動化パイプライン（v2で追加）

### 3-0. [OPS]の位置づけ

[OPS]は社長が現在手作業で行っている代行業務を自動化するパイプラインである。他のパイプライン（DEV/BizDev/RSC/FIN/MKT）が「事業の成長」を目的とするのに対し、[OPS]は「社長の実稼働時間の削減」を直接的な目的とする。

設計原則:
- **労務計算はAIに任せない**: 給与計算・残業計算・休日判定等の法定計算は、Pythonコードで厳密に実装する。AIの幻覚が許されない領域
- **OCRはAIに任せる**: 手書き文字の読み取りはAI（Vision/Claude/Gemini）の得意領域。3段フォールバックで精度を最大化
- **3系統は統一しない**: 信和ミート/KENZAIグループ/介護施設はそれぞれ入力形式・労務ルール・出力形式が異なる。無理に統一せずパラレルに運用する
- **給与らくだ向けCSVが最終出力**: 全系統の最終ゴールは「給与らくだに取り込めるCSVファイル」の自動生成

**対象企業**: 信和ミート

#### KENZAIの拡張設計

新規企業追加時の手順:
1. `KENZAI/[企業名]/` ディレクトリを作成
2. `config.py` に勤務体系（所定労働時間・休憩・締日等）を定義
3. `employees.csv` に社員マスター（氏名・入社日・時給/月給等）を登録
4. 入力形式に応じたパーサーを選択（Excel/PDF/画像）
5. テスト実行 → 差異レポートで検証

**対象**: 特定施設入居者生活介護の介護施設
**頻度**: シフト表は毎月、出勤簿・給与計算も毎月
**設計書**: 再作成が必要

#### 業務フロー

```
[トリガー: 毎月・翌月シフト作成時期]
    │
    ▼
① シフト表作成
    │  shift_exporter.py
    │  入力: 施設の人員配置基準 + スタッフ情報 + 希望休
    │  出力: 月間シフト表（Excel）
    │  ※ 特定施設入居者生活介護の人員配置基準を厳守
    │
    ▼
② シフト表を施設に提出 → 施設が運用
    │
    ▼
③ 月末: 出勤実績を写真で受領
    │  ※ 施設スタッフが手書きで記録した出勤実績の写真
    │
    ▼
④ OCRエンジンで出勤実績を読み取り
    │  第一優先: Mac Vision Framework
    │  第二優先: Claude API
    │  第三優先: Gemini API
    │
    ▼
⑤ シフト表（計画）と出勤実績（実績）を照合
    │  差異の検出（急な欠勤・シフト変更・残業等）
    │
    ▼
⑥ 正式出勤簿の作成
    │  実績ベースの出勤簿をExcelで生成
    │
    ▼
⑦ 労務計算 → 集計
    │  payroll_engine.pyの介護施設版ロジック
    │  ※ 介護施設固有の勤務体系（夜勤・変形労働時間制等）に対応
    │
    ▼
⑧ 給与らくだ向けCSV出力
    │
    ▼
[QA: GPT-5.4 mini]
    │  シフト表 vs 出勤実績の差異チェック
    │  出勤簿の計算結果チェック
    │  人員配置基準の充足確認
    │
    ▼
[CEO承認（LINE ワンタップ）]
    │
    ▼
⑨ 給与らくだへCSVインポート（別Windows PC）
    │
    ▼
[完了]
```
