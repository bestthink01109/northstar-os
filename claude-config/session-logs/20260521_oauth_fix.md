# drive.js OAuth Token Rotation Fix - 2026-05-21

## 問題
別セッションでdrive.jsがinvalid_grantエラーで保存不能になった。

## 根本原因
Google Token Rotationによる短命refresh_token（寿命約5時間）の自動上書き。

drive.jsの`tokens`イベントハンドラ（77行目）が
アクセストークンリフレッシュ時に返ってきた新しいrefresh_tokenを
そのままoauth_tokens.jsonに上書き保存していた。

## 修正内容（drive.js 77-82行目）
```javascript
// 修正前
oauth2Client.on('tokens', (newTokens) => {
    const current = JSON.parse(fs.readFileSync(TOKEN_FILE));
    const merged = { ...current, ...newTokens };
    fs.writeFileSync(TOKEN_FILE, JSON.stringify(merged, null, 2));
});

// 修正後
oauth2Client.on('tokens', (newTokens) => {
    const current = JSON.parse(fs.readFileSync(TOKEN_FILE));
    const merged = { ...current, ...newTokens };
    merged.refresh_token = current.refresh_token; // 元のrefresh_tokenを保持
    fs.writeFileSync(TOKEN_FILE, JSON.stringify(merged, null, 2));
});
```

## 正しいrefresh_token
- 有効: 1//0eVzKd4... (Desktop型・本番公開済み・永続)
- 無効化: 1//0eoWE68... (Token Rotationで発行・約5時間で失効)

## 対応手順
1. oauth_tokens.jsonのrefresh_tokenを正しい値に更新
2. drive.jsを修正してToken Rotation時もrefresh_tokenを保護

## 再発防止
drive.js修正により、今後はToken Rotationが発生しても
oauth_tokens.jsonのrefresh_tokenは永続トークンを維持する。
