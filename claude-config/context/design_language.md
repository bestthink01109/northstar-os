# Design Language | NS-OSV2

更新日: 2026-05-30 セッション2

## 文脈の残し方

- 成果物は Markdown 正本で残す
- latest 面は見やすさのための入口
- 実際の正本は Board / COO notes / archive contexts に置く

## チケット記法

- タスクは Board に戻す
- エラーやズレは remediation_log.md に記録し、Boardチケット化する
- 完了には成果物パスと QA または現物確認の痕跡を残す
- 発展的変更は remediation_log.md の「発展的変更の記録」セクションに明記する

## RSC Daily Report フォーマット（確定）

- 個別レポート（熊本県庁・福岡県庁）を別ファイルで作成（必須）
- Daily_Reportは統合レポート（Section3は個別レポートのサマリー＋参照）
- 正解フォーマット: 熊本県庁トレンドリポート_20260527.md / 福岡県庁トレンドリポート_20260527.md
