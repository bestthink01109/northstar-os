# NorthStar OS — ナレッジベース

> 日々の作業から得られた再利用可能な知識を蓄積・管理するリポジトリ。
> COOセッションで発見したパターン・決定事項・スキル候補をここに集約する。

---

## ディレクトリ構成

```
knowledge/
├── README.md              ← このファイル（全体インデックス・使い方）
├── patterns/              ← 再利用可能な問題解決パターン
│   ├── README.md
│   ├── n8n/               ← n8n固有のパターン集
│   ├── api/               ← API連携パターン集
│   └── ops/               ← OPS業務パターン集
├── decisions/             ← 重要な意思決定の記録
│   ├── README.md
│   └── YYYYMMDD_*.md
└── skills/                ← Claude Codeスキル候補
    ├── README.md
    └── templates/
        └── skill_template.md
```

## 使い方

### 1. パターンを登録する（作業後）

レポートの「💡スキル化候補」セクションで記録されたパターンを `patterns/` に登録する。

登録先の選び方：
| 内容 | 登録先 |
|------|-------|
| n8nノード設定・エラー対処 | `patterns/n8n/README.md` |
| 外部API呼び出し・認証 | `patterns/api/README.md` |
| シフト計算・給与・帳票 | `patterns/ops/README.md` |

フォーマット：
```markdown
## パターン名

- **症状**: （何が起きたか）
- **原因**: （なぜ起きたか）
- **解決策**: （どう直したか）
- **確認日**: YYYY-MM-DD
- **参照**: （関連WF・チケット等）
```

### 2. 決定事項を記録する（重要判断後）

経営・技術・業務に関わる重要な決定は `decisions/YYYYMMDD_タイトル.md` として保存する。
後から「なぜそう決めたか」を参照できるようにする。

### 3. スキル候補を登録する

繰り返し使えるパターンが3回以上登場したら `skills/README.md` にスキル候補として登録する。
テンプレートは `skills/templates/skill_template.md` を使う。

---

## 蓄積ルール

1. **作業後即登録** — セッション終了前に必ずパターン/決定事項を記録する
2. **省略禁止** — 「後で書く」は消滅と同義。必ずその場で書く
3. **具体的に書く** — 「エラーが出た」ではなく「HTTP 400 + 原因 + 解決コード」まで書く
4. **日付を付ける** — すべてのエントリに `確認日: YYYY-MM-DD` を付与する
5. **重複を統合** — 同じパターンが複数回登場したら既存エントリを更新し件数を増やす

## スキル化フロー

```
日々の作業
    ↓
レポートの「💡スキル化候補」に記録
    ↓
knowledge/patterns/ に登録（即時）
    ↓
3回以上登場 → knowledge/skills/ にスキル候補として昇格
    ↓
Claude Code の skills/ に実装 → テスト → 本番運用
```
