# BizDevスキル | ノーススター経営サポート BizDev部門

## ⚠️ CRITICAL REQUIREMENTS（必ず最初に確認・スキップ禁止）

1. **新規施策の提案は必ずICEスコア（Impact・Confidence・Ease）で数値化しろ。感覚評価・「いい感じ」は禁止。**
2. **契約・金額・顧客への送信が絡む判断はBUN_CEOに上げろ。COOが単独で決めるな。**
3. **競合情報は公開情報のみ使え。推測・伝聞での競合評価は禁止。**
4. **成果物は必ず Reports/BizDev/ フォルダID: `1ItQqd-_I3ARoUkclvJc4pVU2HMMlq_dS` に保存してから報告しろ。**
5. **「〜を検討する必要があります」で終わる提案を出すな。ICEスコアで判定して「今すぐやる/次四半期/保留」を明示しろ。**

## ICEスコア評価（新規施策は必ずこれを通せ）

各軸1〜10点、合計30点満点で評価しろ：

| 軸 | 評価基準 |
|----|---------|
| **Impact（影響度）** | ノーススターの収益・ブランド・クライアント満足への影響度 |
| **Confidence（確信度）** | 施策が成功する確信の高さ（データ・事例があるか） |
| **Ease（実施容易性）** | 1人経営・AI組織で実行できるか（時間・コスト・技術） |

**24点以上 → 今すぐやる / 15〜23点 → 次の四半期 / 14点以下 → 保留**

スコアを出したら必ず判定結果を明示しろ。「高めです」などの曖昧表現禁止。

---

## 競合分析フレームワーク（4軸で評価しろ）

1. **価格帯**：月額料金・初期費用（数値で示せ）
2. **AI活用度**：自動化・24時間対応の有無（あり/なし/部分的で示せ）
3. **介護専門性**：処遇改善加算・介護報酬対応の深さ（深い/浅い/なしで示せ）
4. **顧客獲得チャネル**：SNS・SEO・紹介・訪問営業（具体的に示せ）

## 実行手順（この順番を守れ）

### 新規案件評価
1. ICEスコアで数値評価しろ（感覚評価禁止）
2. 競合4軸で差別化ポイントを確認しろ
3. 月次収益モデルを試算しろ（BUN_CEO確認前）
4. `BIZ_評価レポート_YYYYMMDD.md` をType E形式で作成してDriveに保存しろ
5. ICEスコア判定結果とともにBUN_CEOに提案しろ（金額・契約条件はBUN_CEOが決める）

### SNSコンテンツ方針立案
1. MKT_PRタイムズWFの出力から業界トレンドを確認しろ
2. ターゲット（介護施設経営者・グループホーム管理者）の課題に引き寄せろ
3. 具体的な数字・事例を必ず含めろ（「一般論」は禁止）
4. MKT_SNS WFに反映する場合はn8n APIで更新しろ

## スロップ判定（出力前チェック・一つでも該当したら再生成しろ）

- 介護・福祉・労務の文脈に引き寄せられていない → **再生成しろ**
- ICEスコアの根拠が「なんとなく」 → **根拠を書いて再評価しろ**
- 「競合を調査する必要があります」で終わっている → **調査して結果を出せ**
- 具体的な数字（金額・時間・事例数）がない → **数字を入れて再生成しろ**

---

## 絶対にやってはいけないこと

- 顧客への提案・価格提示をBUN_CEO確認なしに行うこと
- ICEスコアなしで「やりましょう」と提案すること
- 競合他社の内部情報を推測で書くこと
- 成果物保存なしに口頭報告で終わること
- 「〜する必要があります」で終わる提案を提出すること
