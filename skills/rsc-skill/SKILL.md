# RSCスキル | ノーススター経営サポート RSC部門

## ⚠️ CRITICAL REQUIREMENTS（必ず最初に確認）

1. **URLから直接コンテンツを取得しろ。要約・印象での出力は禁止。**
2. **BIZ_SCORINGは必ず5軸（市場性・競合優位・実行可能性・緊急性・財務インパクト）で数値化しろ。**
3. **スコア70点以上のみSIGNAL_DBに登録しろ。それ以下は除外して理由を記録しろ。**
4. **成果物は必ず Reports/RSC/ フォルダID: 1I_68Pimq8jKjq6xfPMAeD22oeAHc8mTf に保存してから報告しろ。**

## 実行手順（この順番を守れ）

### Step 1: ソース確認
- 対象URLにアクセスして本文を取得する
- 取得できない場合は「URLアクセス失敗」と記録して次のURLへ進む
- **印象・記憶で補完するな。取得できた内容だけを使え。**

### Step 2: BIZ_SCORING（5軸評価）
以下の基準で各軸0〜20点（合計100点満点）で評価する：

| 軸 | 評価基準 | 満点 |
|----|---------|-----|
| 市場性 | ノーススターのターゲット（介護施設・グループホーム）に直接関係するか | 20 |
| 競合優位 | 他の社労士・コンサルが対応していない情報か | 20 |
| 実行可能性 | 3ヶ月以内にノーススターが活用できるか | 20 |
| 緊急性 | 期限・締切・法改正の施行日が近いか | 20 |
| 財務インパクト | BUN_CEOの収益・コスト・リスクに直接影響するか | 20 |

**判定：70点以上 → SIGNAL_DB登録候補。69点以下 → 除外（理由を記録）**

### Step 3: アウトプット作成
RSCレポートはType E（調査・分析レポート）フォーマットで作成する。
必須セクション：
- ⚡ サマリー（BUN_CEOが30秒で把握）
- 📋 情報詳細（取得した内容の要点）
- 🎯 BIZ_SCORING結果（5軸スコア表）
- 💡 ノーススターへの示唆
- ➡️ 次のアクション

### Step 4: 保存
```bash
node /Users/fuminariaksse/.config/gdrive-mcp/drive.js create \
  "RSC_[内容]_YYYYMMDD.md" \
  1I_68Pimq8jKjq6xfPMAeD22oeAHc8mTf \
  /tmp/rsc_output.md
```

## スロップ判定基準（出力前に確認）

以下の場合はスロップとして再生成する：
- 「一般的に」「多くの場合」「〜と思われます」で終わる文がある

- 固有名詞（施設名・法令名・制度名）がない
- BIZ_SCORINGの数字に根拠が示されていない

## 絶対にやってはいけないこと

- URLにアクセスせずに内容を推測して出力すること → 誤情報リスク
