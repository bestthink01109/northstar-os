## ⚠️ CRITICAL REQUIREMENTS（必ず最初に確認・スキップ禁止）

1. **URLから直接コンテンツを取得しろ。要約・印象・記憶での出力は禁止。取得できない場合は「アクセス失敗」と記録して次へ進め。**
2. **BIZ_SCORINGは5軸（市場性・競合優位・実行可能性・緊急性・財務インパクト）で必ず数値化しろ。感覚スコアは禁止。根拠を1行で示せ。**

5. **成果物は必ず Reports/RSC/ フォルダID: `1I_68Pimq8jKjq6xfPMAeD22oeAHc8mTf` に保存してから報告しろ。**

## 実行手順（この順番を守れ）

### Step 1：ソース取得（印象補完禁止）
- 対象URLにアクセスして本文を取得しろ
- 取得できない場合は「URLアクセス失敗：[URL]」と記録して次のURLへ進め
- **取得できた内容だけを使え。印象・記憶・常識で補完するな。**

### Step 2：BIZ_SCORING（5軸評価・スキップ禁止）

各軸0〜20点（合計100点満点）で評価しろ：

| 軸 | 評価基準 | 満点 |
|----|---------|-----|
| 市場性 | ターゲット（介護施設・グループホーム）に直接関係するか | 20 |
| 競合優位 | 他の社労士・コンサルが対応していない情報か | 20 |
| 実行可能性 | 3ヶ月以内にノーススターが活用できるか | 20 |
| 緊急性 | 期限・締切・法改正の施行日が近いか | 20 |
| 財務インパクト | BUN_CEOの収益・コスト・リスクに直接影響するか | 20 |

**70点以上 → SIGNAL_DB登録候補。69点以下 → 除外（理由を1行で記録しろ）**

### Step 3：アウトプット作成
Type E（調査・分析レポート）フォーマットで作成しろ。
必須セクション（全て揃えるまで完了とみなすな）：
- ⚡ サマリー（BUN_CEOが30秒で把握できる内容）
- 📋 情報詳細（取得した内容の要点）
- 🎯 BIZ_SCORING結果（5軸スコア表・根拠付き）
- 💡 ノーススターへの示唆（具体的なアクションに落とせ）
- ➡️ 次のアクション（「〜する」ではなく「〜しろ」形式で書け）

### Step 4：保存
```bash
node /Users/fuminariaksse/.config/gdrive-mcp/drive.js create \
  "RSC_[内容]_YYYYMMDD.md" \
  1I_68Pimq8jKjq6xfPMAeD22oeAHc8mTf \

```

## スロップ判定（出力前チェック・一つでも該当したら再生成しろ）

- 「一般的に」「多くの場合」「〜と思われます」がある → **削除して断言形式に書き直せ**
- 介護・福祉・ノーススターの文脈に引き寄せられていない → **再生成しろ**
- 固有名詞（施設名・法令名・制度名）がない → **再生成しろ**
- BIZ_SCORINGの数字に根拠がない → **根拠を追記しろ**
- 「〜する必要があります」で終わるアクション → **「〜しろ」に書き直せ**
