## 🔧 v1.3 変更点サマリー（CEO指摘反映・第3弾）

v1.2 に対するCEOの追加指摘4点を受けて、本仕様書は以下を改訂した。

### 変更点1: 新入社員は必ず末尾追加（中間挿入はない）

- **旧仕様（v1.2）**: E-3「新入社員の中間挿入」を要警戒イベントとして扱っていた
- **新仕様（v1.3）**: CEO指示「**新入社員の中間挿入はなく、必ず最後に順に追加される**」を反映
  - E-3 を **削除**
  - E-4「新入社員（末尾追加）」を **標準パターン** に格上げ
  - 対応工数が大幅軽減（indexを末尾に追加するだけ）

### 変更点2: 退職は必ず繰り上がり（一覧から消える）

- **旧仕様（v1.2）**: E-5「一覧から消える場合」とE-6「一覧に残る場合」の2パターンを想定
- **新仕様（v1.3）**: CEO指示「**退職の場合は、必ず繰り上がり**」を反映
  - E-5 を **標準パターン** に確定（繰り上がり必須）
  - E-6「一覧に残るパターン」を **削除**
  - `status = retired` は行削除ではなく履歴として残す（ただし実際の一覧からは消えている）

### 変更点3: コード変更イベント（パート↔社員）を新規追加

- **CEO指示**: 「**コード変更はパートから社員へまた逆もあり**」
- **新仕様**:
  - E-9「雇用区分変更（コード変更）」を新設
  - パート → 社員の昇格時、社員 → パートの変更時の両方に対応
  - `code` / `employment_type` / `teiji` / `pattern` / `chikoku_on` / `break_time` / `yukyu_jikan` を同時に変更する必要あり
  - **給与らくだの一覧順序も変わる可能性が高い** → 手動確定必須

### 変更点4: 手動確認タイミングを「毎月OCR時」に変更（最重要）

- **旧仕様（v1.2）**: 復帰・新入社員・退職イベント発生時のみ手動確認
- **新仕様（v1.3）**: CEO指示「**手動確認は、毎月タイムカードをOCR化した際に確認要**」を反映
  - **毎月必ず `list_capture.py` + `verify_index.py` を実行**
  - OCR化のタイミング（月初）に組み込み、月次処理の一部として定常運用
  - イベント発生時の忘れ防止が抜本的に強化される（予期せぬ並び変化も早期発見）
  - `last_verified` は毎月更新される
  - 結果: index整合性リスクが最小化

### 変更点5: Escキーでの強制ストップ機構を仕様として明示

- **CEO指示**: 「**python auto_input.py には、ESCで強制ストップがかかる仕組みにしてください**」
- **現状**: `context_handover.md` の `start_escape_listener()` で既に実装済み
- **v1.3対応**: §4-11「例外・安全処理」でEscキー機構を **最上位の安全機構** として明記
  - 実装の詳細コード例を§4-13に新設
  - QA基準にもEscキー機構の動作確認を追加
  - エラー対応表でも即時停止の第一選択肢として位置づけ

## 0. パイプライン概要

| 項目 | 内容 |
|:---|:---|
| **パイプラインタグ** | [OPS-A] |
| **部門名** | 代行業務自動化（Operations – Agent A） |
| **クライアント** | 信和ミート（食品関連。介護・福祉ではない） |
| **目的** | 紙のタイムカードから「給与らくだ普及版(BK-2001)」への入力作業を全自動化し、CEO（社労士）の代行業務工数をゼロに近づける |
| **状態** | **開発着手済み（プロトタイプ段階）**。1人分の単独入力は成功、全員一括処理はバグ修正+精度ベンチマーク中 |
| **対象従業員数** | **28名**（退職2名を除外、産休1名を一時除外） |
| **対象月** | 毎月（月次処理） |
| **PM AI** | Claude Sonnet 4.6（通常） / Claude Opus 4.6（法規制判断時） |
| **実行AI / ツール** | **OCRエンジン3段フォールバック**（Mac Vision → Gemini → Claude） + Python（`timecard_to_csv.py`） + pyautogui（`auto_input.py`） |
| **QA AI** | GPT-5.4 mini（OpenAI API、別プロバイダで盲点補完） |
| **QA回数** | 2回（OCR読取後 / 給与らくだ入力前の最終確認）+ CEO承認 |
| **index管理方式** | **手動確定方式（毎月タイムカードOCR時に必ず実施）** |

## 1. 業務フロー詳細

### 1-1. トリガー条件

- **主トリガー**: CEOの手動起動（月初〜月中旬。信和ミートから紙タイムカード画像を受領した時点）
- **副トリガー**: Mac指定フォルダに新規HEIC/JPG画像が一定枚数揃った時点でn8nが検知（Phase 3実装）

### 1-2. 2モジュール構成

本パイプラインは **OS境界をまたぐ2モジュール構成** を採用する。モジュール間は CSVファイル（BOM付きUTF-8） で橋渡しする。

| # | モジュール | 実行環境 | スクリプト | 主機能 |
|:---|:---|:---|:---|:---|
| 1 | OCR & CSV出力 | **macOS** | `timecard_to_csv.py` | HEIC/JPG → **OCR 3段フォールバック** → 絶対ルール適用 → 給与らくだ用CSV |
| 2 | 給与らくだ自動入力 | **Windows 10** | `auto_input.py` | `employees.csv` 動的読込 + CSV → pyautoguiで給与らくだの明細書一覧に全従業員を順番入力 |

## 2. 使用AI / ツール

### 2-1. AIモデルと役割

| レイヤー | 役割 | AI / ツール | 理由 |
|:---|:---|:---|:---|
| PM | 業務仕様の定義、絶対ルールの策定、例外判断 | Claude Sonnet 4.6 | 妥協不可の労務判断が入るため高品質推論 |
| PM（法規制） | 労基法・処遇改善加算など法令判断 | Claude Opus 4.6（自動昇格） | マスター設計書§3のルーティング原則 |
| **実行（OCR 第1選択）** | 紙タイムカードの手書き文字認識（優先） | **Mac Vision Framework** | 無料・ローカル・最速。CEO方針「第1選択はMac」 |
| **実行（OCR 第2選択）** | フォールバック1 | **Gemini API** | 表構造解析に強い。JSON出力安定 |
| **実行（OCR 第3選択）** | フォールバック2 | **Claude API**（Vision対応） | MLLMで手書き解読最強。最後の砦 |
| 実行（CSV変換） | 絶対ルール適用、CSV生成 | Python（`timecard_to_csv.py`） | 法定計算に準ずる決定論的処理はコードで厳密化 |
| 実行（RPA） | 給与らくだへの座標クリック入力 | pyautogui（`auto_input.py`） | UIオートメーション |
| 実行（index検証） | employees.csvと一覧画面の整合性検証 | `verify_index.py`（新規・v1.2） | 手動確定方式のサポート |
| 実行（一覧撮影） | 給与らくだ一覧画面のスクリーンショット取得 | `list_capture.py`（新規・v1.2） | CEOが目視確認するための支援 |
| QA | OCR結果 / 入力結果の照合 | GPT-5.4 mini（OpenAI API） | 別プロバイダで盲点補完 |

### 3-2. employees.csv のカラム定義（v1.3 手動確認タイミング更新）

**カラム定義**:

| カラム名 | 型 | 必須 | 説明 | 例 |
|:---|:---|:---|:---|:---|
| `index` | int | ○ | 給与らくだ一覧での表示順（0始まり。**CEOが給与らくだ画面を毎月確認して確定**） | `0`, `1`, `2` |
| `code` | str | ○ | 給与らくだの社員コード | `社員1`, `パート3`, `N-001` |
| `name` | str | ○ | 氏名（NFC正規化済み） | `矢吹幸子` |
| `employment_type` | str | ○ | 雇用区分: `社員` / `パート` / `業務委託` | `社員` |
| `teiji` | str | ○ | 定時時刻（HH:MM形式） | `8:30`, `9:00`, `13:00` |
| `pattern` | str | ○ | 勤怠パターン A〜E | `A`, `B`, `C`, `D`, `E` |
| `chikoku_on` | bool | ○ | 遅刻早退自動計算 True/False | `True`, `False` |
| `break_time` | str | ○ | 休憩時間帯 | `12:00-13:00` |
| `yukyu_jikan` | str | △ | 有給時間（時給者のみ） | `4:00`, `3:30`, `8:00`, `` |
| `status` | str | ○ | `active` / `retired` / `on_leave` | `active` |
| `last_verified` | date | ○ | **最後に給与らくだ画面で整合性を確認した日付（毎月更新される）** | `2026-04-16` |
| `note` | str | △ | 備考 | `2026年9月頃復職予定` |

**読み込みロジック要点**:
- `status == 'active'` の行のみ処理対象
- `status == 'retired'` / `'on_leave'` は自動スキップ
- **indexはCEOが毎月画面確認で確定** → スクリプトは書き換えない（read-only）
- **last_verified が当月未更新の場合は警告 or 実行拒否**（設定可能）

```python
import csv
import unicodedata
from datetime import date

def load_employees(csv_path, monthly_check=True):
    """employees.csv を動的に読み込み active な従業員のみ返す
    
    v1.3: last_verified の月次チェック機能を強化
    """
    employees = []
    today = date.today()
    current_month = today.strftime('%Y-%m')
    warnings = []
    
    with open(csv_path, encoding='utf-8-sig') as f:
        reader = csv.DictReader(f)
        for row in reader:
            if row['status'] != 'active':
                log.info(f"スキップ（status={row['status']}）: {row['name']}")
                continue
            
            # === v1.3: 当月中に確認済みかチェック ===
            if monthly_check and row.get('last_verified'):
                try:
                    verified = date.fromisoformat(row['last_verified'])
                    verified_month = verified.strftime('%Y-%m')
                    if verified_month != current_month:
                        warnings.append(
                            f"{row['name']}: last_verified が {row['last_verified']} "
                            f"（当月未確認）"
                        )
                except ValueError:
                    warnings.append(f"{row['name']}: last_verified 不正値")
            
            row['name'] = unicodedata.normalize('NFC', row['name'])
            row['index'] = int(row['index'])
            row['chikoku_on'] = row['chikoku_on'].lower() == 'true'
            employees.append(row)
    
    if warnings:
        print("⚠️  当月中にemployees.csvの整合性確認が行われていません:")
        for w in warnings:
            print(f"    {w}")
        print()
        print("    手順:")
        print("    1. 給与らくだで一覧画面を開く")
        print("    2. python list_capture.py で撮影")
        print("    3. python verify_index.py --manual で確認")
        print("    4. 修正・更新後に再度このスクリプトを実行")
        print()
        response = input("このまま続行しますか? [y/N]: ")
        if response.lower() != 'y':
            sys.exit("中断しました（月次確認が未実施）")
    
    return sorted(employees, key=lambda e: e['index'])
```

```python
import sys
import pyautogui

# === pyautogui 安全設定 ===
pyautogui.FAILSAFE = True   # マウス左上で停止
pyautogui.PAUSE = 0.08      # 各操作間の待機

# === Escキー強制停止フラグ ===
STOP_FLAG = False

def start_escape_listener():
    """Escキーで強制停止するリスナーを開始
    
    CEO指示: 「ESCで強制ストップがかかる仕組み」の実装
    """
    global STOP_FLAG
    try:
        import keyboard
        def on_escape(e):
            global STOP_FLAG
            STOP_FLAG = True
            print("\n" + "!" * 60)
            print("  Escキーが押されました。停止シグナルを送信します。")
            print("!" * 60)
        keyboard.on_press_key('esc', on_escape)
        print("✅ Escキー強制停止機構: 有効")
    except ImportError:
        print("⚠️  keyboardモジュール未インストール。")
        print("    Escキー停止は無効です。")
        print("    pip install keyboard でインストールしてください。")
        print("    フォールバック: マウス左上隅でpyautogui FAILSAFEが発動します。")

def check_stop():
    """強制停止フラグを確認し、TrueならsysExit
    
    全てのpyautogui操作の直前に呼び出すこと。
    これによりEsc押下から次の操作までの間に確実に停止する。
    """
    if STOP_FLAG:
        print("\n" + "!" * 60)
        print("  強制停止を実行します。")
        print("  給与らくだの画面状態を確認してください。")
        print("!" * 60)
        sys.exit(0)

def safe_click(pos, pause=0.3):
    """check_stop()を組み込んだ安全なクリック"""
    check_stop()  # ← クリック前に必ずチェック
    pyautogui.click(pos[0], pos[1])
    time.sleep(pause)

# === メインエントリーポイント ===
if __name__ == '__main__':
    # 1. 最初にEscリスナーを起動
    start_escape_listener()
    
    # 2. 本処理開始
    print("5秒後に開始します。停止はEscキー or マウス左上へ。")
    for i in range(5, 0, -1):
        print(f"  {i}...")
        time.sleep(1)
        check_stop()  # カウントダウン中もEsc検知
    
    # 3. メイン処理（全クリックは safe_click で実行）
    ...
```

**CEO確定ルール**:
- 新入社員は **必ず末尾追加**（中間挿入なし）
- 退職は **必ず繰り上がり**（一覧から消える）
- コード変更（パート↔社員）あり
- **毎月必ず手動確認** → 予期せぬ変動も早期発見

##### 【E-1】休業者の復帰（スレスタアニタ復帰を例に）

**状況**: スレスタアニタ（N-004）が産休から復帰。給与らくだ一覧で N-003 バンダリミランの直後（現 index 12 と 13 の間）に表示される。

**手順**: v1.2と同じ（編集前後の図解付き）。立石章一以降15名が index +1 繰り下がり。

**所要時間**: 30分〜1時間

##### 【E-2】新入社員の末尾追加（v1.3 標準化）

**状況**: 新入社員 `社員20` が加入。給与らくだ一覧の末尾に表示される。

```
Step 1: 月次の list_capture.py で新入社員が末尾に追加されていることを確認
    ↓
Step 2: employees.csv に新行を追加（index = 既存最大+1）
    index,code,name,employment_type,teiji,pattern,chikoku_on,break_time,yukyu_jikan,status,last_verified,note
    28,社員20,<氏名>,社員,<定時>,<パターン>,<遅刻>,<休憩>,,active,2026-05-01,新入社員
    ↓
Step 3: 既存27名のindexは変更不要（CEO確定ルールにより末尾追加のみ）
    ↓
Step 4: verify_index.py --manual で追加を確認
    ↓
Step 5: 次回月次処理で本番実行
```

**所要時間**: 5〜10分（情報収集のみ）

##### 【E-5】中途退職（v1.3 繰り上がり必須）

**状況**: 例えば index 10 の小松京子が退職。給与らくだ一覧から行が消える。

```
Step 1: 月次の list_capture.py で退職者がいないことを確認
    ↓
Step 2: employees.csv で退職者の status を active → retired に変更 + index を -1 に変更
    （履歴は残す）
    ↓
Step 3: 退職者より後の全員（今回は17名）の index を -1 ずつ繰り上げ
    ↓
Step 4: 全員の last_verified を本日日付に一括更新
    ↓
Step 5: verify_index.py --manual で検証
    ↓
Step 6: dry-run で確認 → 次回月次処理で本番実行
```

**所要時間**: 15〜30分

##### 【E-9】雇用区分変更（パート↔社員・v1.3 新規）

**状況の例**: 川口紀美子がパート → 社員に昇格。新コード `社員21` が採番される。

```
Step 1: 月次の list_capture.py で変更後の一覧を確認
    → 給与らくだ上で「社員ゾーン」に移動している可能性大
    ↓
Step 2: employees.csv で対象従業員の以下を変更:
    - code: パート3 → 社員21
    - employment_type: パート → 社員
    - pattern: （必要なら変更）
    - chikoku_on: False → True（社員なら基本True）
    - break_time: 13:00-14:00 → 12:00-13:00（通常の社員）
    - yukyu_jikan: 4:00 → （空文字。社員は月給なので不要）
    - index: 一覧での新しい位置に移動
    ↓
Step 3: 影響を受ける他の従業員のindexも連動変更
    ↓
Step 4: 過去のCSVファイル（`パート3_川口紀美子.csv`）と新CSV（`社員21_川口紀美子.csv`）
    の運用切り替えを確認
    ↓
Step 5: 全員の last_verified を本日日付に一括更新
    ↓
Step 6: verify_index.py --manual で検証
    ↓
Step 7: dry-run で確認 → 次回月次処理で本番実行
```

**所要時間**: 30分〜1時間

**重要**: 変更月は法的・給与計算上の重大イベントなので、QA#2で特に念入りに検証すること。

## 8. エラー対応

### 8-1. エラー種別と対処（v1.3 更新）

| エラー種別 | 原因 | 対処 |
|:---|:---|:---|
| OCR第1〜3段失敗 | （v1.1と同じ） | 順次フォールバック |
| OCR全段失敗 | 全エンジン失敗 | 画像を `errors/` 隔離 + CEO通知 |
| 社員マスター不一致 | employees.csvに該当従業員なし | 警告ログ出力 → スキップ |
| employees.csv 読込失敗 | ファイル欠損・壊れたCSV | **即停止** → CEO通知 |
| 画面解像度変更 | Windows PCの表示設定変更 | 座標全取り直し |
| 給与らくだのダイアログ出現 | 入力値エラー・重複データ等 | 即停止 → CEOが画面確認 |
| CSV BOM欠落 | 編集時にBOMを失った | `utf-8-sig` で再保存 |
| NFD濁点分離 | Mac由来のファイル名 | `unicodedata.normalize('NFC')` |
| Page Down境界誤り | 給与らくだのバージョン違い | index // 14 の係数を再調整 |
| **last_verified 当月未更新** | **手動確認がまだ** | **実行前警告 → CEO承認プロンプト or 中断** |
| index変動を検出せず実行 | E-1/E-5/E-9のイベント発生を見逃し | **毎月の手動確認で事前検出** |
| **Escキー押下** | **CEOが異常を検知して停止** | **STOP_FLAG=True → 即停止（v1.3で明示化）** |
| **マウス左上検知** | **フェイルセーフ発動** | **pyautogui FailSafeException → 即停止** |
| **keyboard モジュール未インストール** | **Esc停止機構の無効化** | **警告メッセージ表示 → マウス左上フォールバックで続行** |
