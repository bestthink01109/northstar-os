# LINE自動化構築サービス パッケージ作成
- layer: 3
- complexity: high
- type: new-product

## 背景
ノーススター経営サポートが販売するLINE自動化構築サービスの雛型パッケージを作成する。
自社で動かしているLINE×n8nの仕組みを、外部クライアント向けにテンプレート化する。

## 成果物

### 1. n8n WFテンプレート3本（JSON）
以下をJSON形式で作成し `/root/northstar-os/dev/line-service/` に保存する。

**WF-A: キーワード自動応答**
- LINE Webhookでメッセージ受信
- キーワードマッチング（設定ファイルから読み込み）
- マッチしたキーワードの返信文を送信
- 設定ファイル: `keywords.json` に `[{"keyword":"予約","reply":"予約はこちら..."}]` 形式

**WF-B: ウェルカムメッセージ**
- 友達追加イベント（follow）を受信
- 設定ファイルから歓迎メッセージを取得
- 即時送信

**WF-C: 定期一斉通知**
- Cron設定（曜日・時刻を設定ファイルで指定）
- Google DriveまたはGoogleスプレッドシートから友達IDリストを取得（オプション）
- LINE Broadcast APIで一斉送信

### 2. 設定ファイル仕様書
クライアントが変更する設定ファイルの仕様を記述した `CONFIG_GUIDE.md`

### 3. 導入手順書
`SETUP_GUIDE.md` - n8n設定・LINE Messaging API連携の手順（スクリーンショット不要・テキストのみ）

### 4. デプロイスクリプト
`deploy.sh` - 新規クライアント向けにWFを自動インポートするスクリプト

## 技術仕様
- n8n REST APIを使用してWFをインポート
- LINE Messaging API（Webhook・Push API）
- 設定ファイルはJSON形式（n8nのEnvironment Variables or ファイル読み込み）

## 出力先
`/root/northstar-os/dev/line-service/`

## 完了条件
- 3本のWF JSONが動作する形で存在する
- CONFIG_GUIDE.md・SETUP_GUIDE.md が作成されている
- deploy.sh が実行可能

## Codex引き渡しメモ
- 既存のLINE Harnessの実装（northstar-line.bestthink01109.workers.dev）を参考にすること
- n8n APIのクレデンシャル参照パターンはBizDevスキャンWFと同じ形式で
- テストは実際にn8nにインポートして動作確認すること
