# KENZAI SS 会社設定タブ追加（外販対応）
- layer: 3
- complexity: high
- type: python

## 背景
KENZAIシステムを外販するにあたり、勤務条件がハードコードされている問題を解消する。
現在は建設業専用（7h×月〜金、5h×土）だが、クライアントごとに設定変更できる仕組みが必要。

## 対象ファイル
```
/Users/fuminariaksse/Desktop/antigravity Folder/company/Development/KENZAI/
├── hirano/config.py       ← ハードコード値の参考
├── junsei/config.py       ← 同上
├── base_config.py         ← CompanyConfig定義
└── parsers/excel_parser.py
```

## 実装内容

### 1. Googleスプレッドシートへの「会社設定」シート追加

各クライアント用スプレッドシートに「会社設定」シートを追加する。
以下のセルを設ける：

```
A列：設定項目名
B列：設定値（クライアントが入力）

B2: 労働時間制の種類（通常 / 月単位変形 / 年単位変形）
B3: 平日所定労働時間（時間）例：7 or 8
B4: 土曜所定労働時間（時間・なしは0）例：5 or 0
B5: 日曜労働（あり/なし）
B6: 週法定労働時間上限（通常：40）
B7: 半日の基準時間（例：3.5 or 4）
B8: 月単位変形・月所定労働時間（通常制の場合は空欄）
B9: 月単位変形・1日上限時間（法定10h以内）
B10: 月単位変形・週上限時間（法定52h以内）
```

### 2. base_config.py の CompanyConfig拡張

以下のフィールドを追加：
```python
labor_system: str  # "standard" / "monthly_variable" / "annual_variable"
monthly_scheduled_hours: Optional[float]  # 月単位変形時の月所定時間
daily_max_hours: float = 10.0  # 日の上限（変形時）
weekly_max_hours: float = 40.0  # 週の上限（通常40、変形52まで）
```

### 3. スプレッドシートから設定を読み込む仕組み

新規ファイル `parsers/ss_config_reader.py` を作成：
- Googleスプレッドシートの「会社設定」シートを読み込む
- CompanyConfigオブジェクトに変換して返す
- 設定値のバリデーション（上限チェック等）

### 4. main.py の対応

`--company` 引数でconfig.pyではなくSSから設定を読む選択肢を追加：
```bash
python3 main.py --company hirano --month 202605  # 従来通りconfig.pyを使用
python3 main.py --ss-config <spreadsheet_id> --month 202605  # SSから設定読み込み
```

## 完了条件
- 「会社設定」シートに値を入れてPythonを実行すると、その設定で計算できる
- 建設業（7h×5日）・通常業種（8h×5日）の両方で正常動作確認
- バリデーションエラー（週40h超など法令違反設定）を検出してエラー出力

## 注意事項
- 月単位・年単位変形労働時間制の計算ロジックは別チケット（仕様確定待ち）
- Excelを保存しないルールは引き続き遵守（読み取り専用）
- openpyxlとgspreadの両方を考慮すること

## Codex引き渡しメモ
- CompanyConfigの拡張は型安全に行うこと
- SSからの読み込みは認証情報エラーを適切にハンドリング
- テストはhirano社の実データで行うこと
